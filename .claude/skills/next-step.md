# Next Step Skill

## 用途
根据项目进度（PATH.md）和当前焦点（NOW.md）规划用户下一步学习内容。

## 工作流程

### 1. 读取进度
```
1. 读取 PATH.md 获取完整知识地图
2. 读取 NOW.md 获取当前焦点
3. 确定用户当前位置（算子线 + 理论线）
```

### 2. 分析状态
```
- 算子线：A/B/C/D/E 哪个阶段？
- 理论线：6个子类哪个主题？
- 是否卡在某个具体问题上？
```

### 3. 生成建议
```
📍 当前位置：[算子线 A3 / 理论线 online softmax]

🎯 算子线接下来：
- [ ] [具体任务]
- 资源：[lessons/notes/reference 链接]
- 验收：[如何算完成]

🧠 理论线接下来：
- [ ] [主题]
- 资源：[papers/notes 链接]
- 产出：[预期输出]

💡 选择建议：
- 如果你精力好 → 建议 [具体建议]
- 如果时间少 → 建议 [简化路径]
- 如果想调节奏 → [替代方案]
```

### 4. 用户确认
```
等待用户选择，不强制推进。
```

## 规则
- **拉模式**：用户问"接下来做什么"才调用
- **权威源**：永远以 PATH.md 为准，不臆测进度
- **双线并行**：算子线 + 理论线各给一个建议
- **可调节奏**：提供多种选项，用户选择

## 特殊情况处理
- 用户卡在具体问题 → 调用 code-review 或 concept-explain
- 用户想跳过某部分 → 说明影响，尊重选择
- 用户想换方向 → 提供 PATH.md 的其他路径

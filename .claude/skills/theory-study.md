# Theory Study Skill

## 用途
指导理论线（6个子类）学习，帮助用户产出一页笔记。

## 理论线结构

### 6 大子类
```
1. GPU 优化算法
2. 量化
3. 注意力演进
4. 模型架构
5. 推理系统技术
6. 训练/并行
```

### 学习流程
```
1. 读论文/资料（30-60分钟）
2. 提炼核心机制（一页笔记）
3. 理解"为什么"和"怎么实现"
4. 存入 notes/algorithms/
5. [可选] 精读论文存入 papers/
```

### 笔记模板
```markdown
# [主题]

## 核心机制
[一句话概括]

## 为什么需要
[解决的问题、痛点]

## 怎么实现
[关键技术、算法]

## 与其他方案对比
[表格形式]

## 业界应用
[哪些模型/框架用了]

## 参考文献
[论文链接、博客等]
```

## 6 大子类指导

### 1. GPU 优化算法
```
示例：online softmax, parallel reduce, prefix sum
学习重点：算法本身的并行化技巧
验收标准：能推公式，能讲为什么比朴素方法好
```

### 2. 量化
```
示例：INT8/FP8 格式, AWQ, GPTQ, SmoothQuant
学习重点：数值精度、量化方法、校准策略
验收标准：能解释 INT8 vs FP8 优劣，能选方案
```

### 3. 注意力演进
```
示例：MHA→MQA→GQA→MLA, Flash Attention 1/2/3
学习重点：复杂度降低、工程优化、KV 压缩
验收标准：能画演进图，能讲 trade-off
```

### 4. 模型架构
```
示例：MoE, Mamba/SSM
学习重点：架构创新、推理挑战
验收标准：能讲动机和关键技术
```

### 5. 推理系统技术
```
示例：continuous batching, PD 分离,投机解码
学习重点：系统级优化、调度策略
验收标准：能讲清解决了什么问题
```

### 6. 训练/并行
```
示例：ZeRO/FSDP, TP/PP/EP
学习重点：通信策略、显存优化
验收标准：能画通信图
```

# Weekly Report Skill

## 用途
基于 git 活动自动生成周报，记录本周完成的工作。

## 工作流程

### 1. 收集信息
```
git log --since="1 week ago" --oneline
git diff --stat
文件修改时间分析
```

### 2. 分类整理
```
算子线进展：[具体完成项]
理论线进展：[具体完成项]
代码提交：[commit 摘要]
新增笔记：[notes/algorithms/ 新增]
```

### 3. 生成周报
```markdown
# Week X - [日期范围]

## 算子线
✅ [完成项1]
✅ [完成项2]
🚧 [进行中]

## 理论线
✅ [主题] - [笔记链接]
📖 [阅读中]

## 代码提交
- [commit1] ([文件])
- [commit2] ([文件])

## 问题与解决
- [问题] → [解决]

## 下周计划
- [计划1]
- [计划2]
```

## 输出格式
```
📊 本周活动总结：

算子线：
✅ [完成项]
🚧 [进行项]

理论线：
✅ [完成项]
📖 [进行项]

📝 周报已保存：weekly/YYYY-MMDD-weekX.md
```

## 调用时机
- 用户说"这周结束了，生成周报"时
- 用户完成一个阶段需要记录时
- 每周定期回顾时

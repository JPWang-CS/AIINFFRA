# 回顾周报 — 生成格式

> 这不是你填的模板，是**我自动生成回顾时遵循的格式**。
> 触发：你说"X 学完了" → 我读 `git log`（提交/修改时间、改了哪些文件）→ 按下面格式写一篇。
> 命名：`YYYY-MM-DD-<主题>.md`（如 `2026-06-24-gemm-tiled.md`）。

---

## 模板

```markdown
# <主题> — <日期范围>

> 算子线/理论线 · 对应 PATH 的 <阶段，如 A3>

## 做了什么
（根据 git 提交时间线 + 改动文件，列实际完成的事）
- ...

## 关键数据
（GFLOPS、bandwidth、正确性、提速倍数等可量化的）
- ...

## 卡点 / 怎么解决的
（遇到的问题 + 解法，对复盘和面试有用）
- ...

## 面试可用点
（这次学到的、能写进简历或答面试的）
- ...

## 下一步
（接 NOW 的"接下来"）
- ...
```

---

*这些回顾是事后记录，对求职叙事和复盘有用。教程内容在 [lessons/](../lessons/)，进度在 [PATH.md](../PATH.md)。*
